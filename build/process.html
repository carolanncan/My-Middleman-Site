Middleman

When I was looking over the list Dan provided, Middleman was actually where I wanted to start. I had previously completed some CodeAcademy tutorials and spent a few hours looking over Sass.


The initial setup was the hardest part of the process for me. I spent about 2-3 hours working to get my files set up correctly and to understand some of the nuances of the file structure. There are definitely a few simple things I wish I had known going in, and a few things I still want to learn more about later.


My favorite feature of Middleman, so far, is being able to pull in content dynamically from other layouts.

  • Middleman Forum
  • Ben Frain, Understanding Middleman the Static Site Generator for Faster Prototyping

Bash

I could probably count on my hand the amount of times I've used my command line - including watching ASCII Star Wars. So installing things from a bash shell, navigating through directories, remembering what directory I'm in before I install something... all took a little getting used to. But I really enjoyed becoming one with the command line.

Sass

Sass was a lot of fun to learn! After building one site from scratch with only CSS, I immediately had an appreciation for the cool things SASS offers and how easy it can be to apply them.

Bourbon

There are some elements in Bourbon that I can see being most useful for large projects, or for a team of people building something together. For my practice site, it has been a little bit of a stretch to realize the full benefit of using some of the mixins, for example. But the more I play around with it, the more I really like using it. ...And Bourbon Neat is 2x the fun.
